Vessel Trajectory Prediction Project

Exploring vessel movements in the Saronic Bay using real AIS data

Project Overview

This project focuses on analyzing and predicting vessel trajectories using real-world data collected from the Automatic Identification System (AIS) antenna located at the University of Piraeus. The primary objective is to enhance our understanding of vessel movements in the Saronic Bay area and develop predictive models that can forecast ship trajectories based on historical AIS data.

Database Setup & Data Collection

The first step was to establish a spatial database called thalassaDB to store raw AIS data. The data tracked vessels in the Saronic Bay area, focusing on passenger vessels visiting Piraeus Port. The AIS data includes vessel IDs, locations, timestamps, speed and other movement details, making it ideal for trajectory prediction.

    Data Filtering: Filtered out irrelevant data, focusing on vessels that are in motion to ensure accurate analysis.

Data Cleaning & Feature Engineering

Data cleaning was a crucial phase in the project, addressing errors, inconsistencies, and missing values in the raw AIS dataset.

This phase involved several key tasks:

1. Temporal and Speed Consistency

The initial step focused on ensuring that the dataset maintained consistent temporal and speed values. Sudden fluctuations in speed were often indicative of GPS errors, so rolling averages were applied to smooth out these values. Records with speeds exceeding the maximum allowable limits for each vessel type were filtered out to maintain realistic behavior.


2. Spatial Cleaning with R-tree Indexing

To ensure the spatial accuracy of vessel positions, R-tree spatial indexing was employed. This technique helped filter out erroneous data points, such as instances where a vessel appeared to travel unrealistic distances in a single timestamp. Spatial joins with port locations were also performed to confirm that vessels were appropriately located near ports when expected.


3. Feature Engineering Based on Vessel Status Classification

The dataset was enriched by categorizing vessel statuses (e.g., at anchor, underway using engine, engaged in fishing) based on operational conditions. This classification was determined using vessel speed, type, and port associations. The data was then divided into trips and trajectories, with trips defined by key status changes and trajectories tracking movements between these changes.


4. Handling Missing Data & Converting Trajectories

After cleaning the data and trajectories creation, we encountered missing positions in the AIS dataset, which were common in areas with weak GPS signals or due to transmission errors. To address this, we used linear interpolation to fill in the gaps between existing data points. This process ensured that vessel trajectories were continuous and accurately represented. By performing interpolation at 30-second intervals, we created a uniform dataset that could be effectively used for model training. This step effectively transformed each vessel's trajectory into a time series format, where data points were organized chronologically.

Mini-Trajectories & Feature Preparation

To optimize model training, continuous vessel trajectories were split into mini-trajectories. Each mini-trajectory represented a small time window and captured both short-term and long-term movement patterns. This step was crucial in improving the efficiency and accuracy of the neural network models.


Each mini-trajectory consisted of 14 time steps, which was a suitable trade-off between capturing meaningful movement patterns and reducing the complexity of the dataset. Additionally, the features were normalized using Min-Max scaling to ensure that all features contributed equally to the model.

×

Model Development & Training

We chose a Gated Recurrent Unit (GRU) model, a type of Recurrent Neural Network (RNN), for its ability to capture temporal dependencies in sequential data.

The model was designed to process input sequences of vessel positions (longitude and latitude) along with features like distance from the previous signal and course direction. We used sliding windows of 14 time steps, enabling the model to learn from both past and future vessel movements.

×

Model Training & Evaluation

The model was trained with a batch size of 16 for 300 epochs.

After training the LSTM model, we evaluated its performance using Root Mean Squared Error (RMSE), a common metric for regression tasks. The model achieved an RMSE of 23.21 meters, indicating that, on average, the predicted vessel positions were within 23.21 meters of the true positions.
