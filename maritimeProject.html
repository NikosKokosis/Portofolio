Vessel Trajectory Prediction Project

Project Overview

This project focuses on analyzing and predicting vessel trajectories using real-world data collected from the Automatic Identification System (AIS) antenna located at the University of Piraeus. The primary objective is to enhance our understanding of vessel movements in the Saronic Bay area and develop predictive models that can forecast ship trajectories based on historical AIS data. This process is split into five main phases: Database Setup and Data Collection, Data Exploration and Queries, Data Cleaning and Feature Engineering, Model Development, and Model Evaluation.

The first step was to establish a spatial database to store the raw AIS data that we would be working with. This database, called thalassaDB, was designed to handle large volumes of vessel data and enable efficient spatial queries for analysis. AIS data, which includes vessel position, speed, course, and timestamp, was collected from the AIS antenna located at the University of Piraeus.

In this phase, we also focused on velocity filtering, ensuring that only vessels in motion were considered for analysis. This step was important because we wanted to eliminate data related to stationary vessels or erroneous GPS readings.

[An image of the database structure]

With the data collected and stored in the database, we began exploring it through SQL queries. This phase involved performing basic queries to explore the vessel information, such as:

  • Vessel ID and names
  • Speed and course of movement
  • Distances between vessels and ports
  • Port entry and exit times

We also manually added additional data like port locations and geometries, and performed geospatial queries to extract insights from spatial relationships between vessels and ports.

Example Questions Answered: "Which vessels are currently within a 10 km radius of Piraeus Port?" or "What is the average speed of vessels within a certain distance from the shore?"

This was the most complex and critical phase, where we addressed errors, inconsistencies, and missing values in the raw AIS data. Key tasks included:

  1. Temporal and Speed Consistency: We applied rolling averages to smooth the speed data and removed records where the speed exceeded realistic limits for each vessel type.
  2. Spatial Cleaning with R-tree Indexing: Using R-tree indexing, we filtered out erroneous spatial data points (e.g., vessels traveling several hundred kilometers within a single timestamp).
  3. Feature Engineering: Vessel statuses were categorized based on operational conditions like speed and type, and the data was split into trips and trajectories.

We also handled missing data through linear interpolation to ensure continuous vessel trajectories.

We converted the vessel trajectories into time series data and broke them into mini-trajectories consisting of 14 time steps. This segmentation helped the neural network capture both short-term and long-term vessel movement patterns.

We also performed Min-Max normalization on the key features (Longitude and Latitude) to improve model efficiency.

We developed a Recurrent Neural Network (RNN) using Long Short-Term Memory (LSTM) architecture to predict future vessel positions based on historical data.

[Image of the model architecture]

The model was trained using a dataset of cleaned and preprocessed AIS data. The final model achieved an RMSE of 23.21 meters, demonstrating high accuracy in predicting vessel positions.

This project successfully demonstrated the power of combining data cleaning, spatial algorithms, and predictive modeling to analyze and forecast vessel trajectories. The model can be used to improve maritime safety, optimize vessel routing, and enhance decision-making in the maritime industry.
